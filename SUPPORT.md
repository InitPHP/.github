# Getting Support

InitPHP is a community-maintained open source project. We do not currently offer paid or SLA-backed commercial support. The channels below cover every legitimate way to get help, in the order you should try them.

## Choose the right channel

Before opening anything, please skim the relevant package's README — most questions are answered there.

| You want to… | Use |
| --- | --- |
| Ask "how do I do X?" | [**GitHub Discussions → Q&A**](https://github.com/orgs/InitPHP/discussions/categories/q-a) |
| Report a bug | The **Issues** tab on the affected package's repository |
| Request a feature or new package | [**GitHub Discussions → Ideas**](https://github.com/orgs/InitPHP/discussions/categories/ideas) — promote to an issue once there is consensus |
| Share something you built with InitPHP | [**GitHub Discussions → Show and tell**](https://github.com/orgs/InitPHP/discussions/categories/show-and-tell) |
| Report a security vulnerability | **Do NOT open a public issue.** Follow [`SECURITY.md`](./SECURITY.md). |
| Report Code of Conduct violations | Email [info@muhammetsafak.com.tr](mailto:info@muhammetsafak.com.tr) — see [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md). |
| Find official documentation | [**initphp.org**](https://initphp.org) |
| Browse releases | [**Packagist**](https://packagist.org/packages/initphp/) |

## What we ask of you

When opening an issue or discussion, please:

- **Search first.** A quick search across Issues and Discussions often finds an existing answer.
- **Provide a minimal reproducer.** "It doesn't work" is hard to act on; a 10-line script that demonstrates the problem is gold.
- **Tell us your environment.** PHP version, OS, the package version (`composer show initphp/<package>`), and any relevant extensions.
- **Be patient.** This is a small-team project; response times vary. If you need a guaranteed turnaround, see *Commercial support* below.

## Commercial support

There is currently no commercial support offering for InitPHP. If your organisation needs a paid engagement (consulting, integration help, custom features, prioritised bug fixes), reach out at [info@muhammetsafak.com.tr](mailto:info@muhammetsafak.com.tr) and we can discuss whether something can be arranged on a case-by-case basis.

## Contributing back

If you found a bug and have a fix, or you have an improvement to share, please open a Pull Request — see [`CONTRIBUTING.md`](./CONTRIBUTING.md). Contributions are the single biggest way to "support" the project.
